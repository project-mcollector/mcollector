# Contributing

## Pull Requests Workflow

1. Create a branch from `main`
2. Make your changes
3. Squash commits into one before opening PR
4. PR title must follow conventional commits:

```text
<type>: <description> #issue
```

**Types:** `feat`, `fix`, `chore`, `docs`, `refactor`, `test`

**Examples:**

```text
feat: add user authentication #52
fix: resolve login redirect issue #67
chore: update dependencies #69
```
